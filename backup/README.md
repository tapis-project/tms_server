# TMS Server DB backups

This directory contains the script to create a backup of the TMS server DB and push it to s3.

During installation of TMS the script is copied to `~tms/backups/scripts`. Backup files generated by the script
are located in directory `~tms/backups/tms`.

NOTE: If default installation directories are not used for TMS, please update TMS_ROOT and TMS_HOME in the script.

The script supports backup for two types of DB, postgres and sqlite. For TMS 0.2.0 and earlier use SQLITE.
For 0.3.0 and later use POSTGRES.

For POSTGRES the file ~tms/.tms/local/tms-db-env must contain the DB configuration, for example:
```
PG_USER=tms
PG_DBNAME=tmsdb
PG_HOST="localhost"
PG_PORT="5432"
PG_PASSWORD="*******"
PG_DEPLOYMENT="tms-postgres"
```

Typically, the script will be run as a cron job. Here is an example crontab entry that can be used to run the script
at 4am every day:
```
# 4 0 * * * /home/tms/tms_server/backups/scripts/backup_tms_server.sh
```
